# 枣强中医院众阳接口现场测试指南（统一版）

> **租户**：`zaoqiang-tcm-001`（枣强县中医院）  
> **对接名称**：SPD对接  
> **依据文档**：接口文档1（科室/人员）、接口文档2（SPD 药品材料）  
> **当前环境**：正式库 `prod` — `https://zqxzyyy.msuncloud.cn`  
> **本阶段**：仅查询、查看回参，**不落库**；推送 2.5.41/2.5.42 **未开放**

---

## 1. 推荐方式：统一联调页（实施人员首选）

1. 启动 scminterface（默认端口 **8088**），确认 `application.yml` 中 `msun.enabled=true`、`active-env=prod`
2. 浏览器登录：`http://<前置机IP>:8088/login.html`（`admin` / `admin123`）
3. 打开统一联调页：**主页 → 枣强众阳联调**，或直接访问  
   **`http://<前置机IP>:8088/zaoqiang-msun-probe.html`**

### 页面功能

| 区域 | 说明 |
|------|------|
| **环境与清单** | 环境信息、一键顺序测试、测试清单、**保存参数到本地** |
| **科室/人员** | 2.1.9、2.1.12 — 按文档入参列表生成表单（含下拉、说明、必填*） |
| **SPD 查询** | 2.5.44 / 58 / 62 / 63 / 43 / 102 — 完整入参可编辑 |
| **高级 JSON** | 各接口可展开 JSON 编辑器，覆盖表单或编辑 POST 请求体 |
| **右侧回参** | 当前回参、测试记录、导出/复制反馈 |

每个接口支持：**调用**、**重置**、**表单→JSON**、**JSON→表单**；`hospitalId`/`orgId` 留空则用服务端正式库配置。

测试完成后 **生成反馈 → 复制反馈** 发给项目组。

## 2. 接口一览

### 2.1 基础字典（接口文档1）

| 编号 | 名称 | 方法 | 众阳路径 | 联调页/HTTP 路径 |
|------|------|------|----------|------------------|
| 2.1.9 | 科室基本信息 | GET | `/msun-middle-base-common/v1/depts` | `/msun/depts` |
| 2.1.12 | 用户身份信息 | GET | `/msun-middle-base-common/v1/identities` | `/msun/identities` |

人员身份：`roleType` / `deptId` / `identityId` / `userId` **至少传一个**。

### 2.2 SPD 查询（接口文档2）

| 编号 | 名称 | 方法 | 众阳路径 | 联调页/HTTP 路径 | 状态 |
|------|------|------|----------|------------------|------|
| 2.5.44 | 药品、材料字典 | GET | `/msun-middle-base-resource/v1/drug-dict-infos` | `/msun/spd/query/drug-dict-infos` | 已实现 |
| 2.5.58 | 分类字典 | GET | `/msun-middle-base-dict/v1/dict-category` | `/msun/spd/query/dict-category` | 已实现 |
| 2.5.62 | 供应商 | GET | `/msun-middle-base-dict/v1/drug-supplieres` | `/msun/spd/query/drug-suppliers` | 已实现 |
| 2.5.63 | 生产厂商 | GET | `/msun-middle-base-dict/v1/drug-produceres` | `/msun/spd/query/drug-producers` | 已实现 |
| 2.5.43 | 药房批次库存 | GET | `/msun-middle-base-resource/v1/drug-batch-stocks` | `/msun/spd/query/drug-batch-stocks` | 已实现 |
| 2.5.102 | 一级库入退库记录 | POST | `/msun-middle-base-resource/v1/query-yk-instock` | `/msun/spd/query/yk-instock` | 已实现 |

### 2.3 参数依赖（易错）

| 接口 | 说明 |
|------|------|
| 2.5.43 | `deptId`←2.1.9；`drugId`、`drugSpecPackingId`←2.5.44，须成对有效 |
| 2.5.102 | `startTime`、`endTime` 必填，格式 `yyyy-MM-dd HH:mm:ss` |
| 2.5.58 | 参数名 `keyWord`（非 Keyword） |
| 2.5.62/63 | `materialOrDrug` 文档有 1/2 差异，以回参为准 |

## 4. HTTP API 参考（可选，需 Token）

基址：`http://<IP>:8088/api/customer/zaoqiang-tcm-001/msun`

```http
POST /login
{"username":"admin","password":"admin123"}
```

```powershell
$base = "http://127.0.0.1:8088"
$login = Invoke-RestMethod -Method Post -Uri "$base/login" -ContentType "application/json" `
  -Body '{"username":"admin","password":"admin123"}'
$h = @{ Authorization = "Bearer $($login.data.token)" }

Invoke-RestMethod -Headers $h -Uri "$base/api/customer/zaoqiang-tcm-001/msun/env"
Invoke-RestMethod -Headers $h -Uri "$base/api/customer/zaoqiang-tcm-001/msun/depts?invalidFlag=-1"
Invoke-RestMethod -Headers $h -Uri "$base/api/customer/zaoqiang-tcm-001/msun/spd/query/drug-dict-infos?limitCount=10&materialOrDrug=0"
```

回参重点看 **`data.hisBody`**：`success`、`code`、`message`、`data`。

## 7. 常见问题

| 现象 | 处理 |
|------|------|
| `openapi@9984` | 联系众阳为 appId 开通对应接口权限 |
| `success: true`, `code: 0000` | 正常，导出反馈或截图 JSON |
| 401 | 重新登录 |
| 404 | 确认 `msun.enabled=true` 且已重启 |
| 2.5.43 失败 | 检查三 ID 是否来自 2.1.9 + 2.5.44 有效对照 |
| 正式失败、灰度成功 | 确认 `active-env=prod` 且 hospitalId/orgId 与正式成套 |
