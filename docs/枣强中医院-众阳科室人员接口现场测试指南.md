> **租户**：`zaoqiang-tcm-001`（枣强县中医院）  
> **对接名称**：SPD对接  
> **测试目的**：调用众阳 HIS 标准接口，查看**科室（2.1.9）**与**人员身份（2.1.12）**回参结构，本阶段**不做落库**。

## 1. 测试范围

| 接口编号 | 名称 | 方法 | 路径 |
|----------|------|------|------|
| 2.1.9 | 科室基本信息 | GET | `/msun-middle-base-common/v1/depts` |

## 3. 方式一：本地/IDE 直连测试（推荐先做）

无需启动 scminterface 服务，直接运行 Java 主类，控制台打印完整 JSON 回参。

### 3.1 前置条件

- 已安装 **JDK 8**、**Maven**
- 工程路径：`scminterface`（多模块项目）
- 运行机可访问对应众阳域名（灰度或正式）

### 3.2 编译（首次或代码变更后）

在项目根目录 `scminterface` 下执行：

```powershell
mvn install -pl scminterface-common,scminterface-framework -am -DskipTests
```

### 3.3 运行

**A. Cursor / VS Code**

在「运行和调试」中选择：

| 配置名称 | 环境 |
|----------|------|
| **枣强众阳探针-测试库** | 灰度 `test`（默认） |
| **枣强众阳探针-正式库** | 正式 `prod` |

主类：`com.scminterface.customer.zaoqiangTcm.msun.test.ZaoqiangTcmMsunProbeMain`

**B. 命令行**

```powershell
cd <scminterface 根目录>
# 测试库
mvn -pl scminterface-framework org.codehaus.mojo:exec-maven-plugin:3.1.0:java `
  "-Dexec.mainClass=com.scminterface.customer.zaoqiangTcm.msun.test.ZaoqiangTcmMsunProbeMain"

# 正式库（末尾加 prod）
mvn -pl scminterface-framework org.codehaus.mojo:exec-maven-plugin:3.1.0:java `
  "-Dexec.mainClass=com.scminterface.customer.zaoqiangTcm.msun.test.ZaoqiangTcmMsunProbeMain" `
  "-Dexec.args=prod"
```

### 3.4 预期控制台输出

1. 环境横幅：`租户`、`baseUrl`、`appId`、`hospitalId`、`orgId`
2. 科室接口：标题 `2.1.9 科室基本信息`，格式化 JSON
3. 人员身份接口：标题 `2.1.12 用户身份信息`，格式化 JSON
4. 每段回参附带：`success`、`code`、`message`、`data.size`（若有 data）

### 3.5 判定标准

| 结果 | 含义 | 现场处理 |
|------|------|----------|
| `success: true` 且 `code: 0000`（或文档约定成功码） | 接口与权限正常 | 截图/保存 JSON，交开发做落库设计 |
| `success: false`，`code: openapi@9984` | **无接口访问权限** | 联系众阳实施，为当前 `appId` 开通 2.1.9、2.1.12 |
| 连接超时 / SSL 错误 | 网络或防火墙 | 检查前置机出网、域名白名单、端口 9443（灰度） |
| 签名相关错误 | 密钥或时间戳 | 核对 `appSecret` 是否为 SM2 十六进制；确认服务器时间准确 |

## 4. 方式二：通过 scminterface HTTP 探针（模拟前置机调用）

适合验证「服务启动 + Token + 接口路径」是否与生产部署一致。

### 4.1 开启探针

编辑 `scminterface-admin/src/main/resources/application.yml`：

```yaml
scminterface:
  customer:
    zaoqiang-tcm-001:
      msun:
        enabled: true          # 已默认启用
        active-env: prod       # 已切换枣强正式库
```

修改后**重启** scminterface 服务（默认端口 **8088**）。

### 4.2 登录获取 Token

```http
POST http://<前置机IP>:8088/login
Content-Type: application/json

{
  "username": "admin",
  "password": "admin123"
}
```

从响应 `data.token` 取出令牌。后续请求在 Header 中携带：

```http
Authorization: Bearer <token>
```

> 探针路径**未加入白名单**，必须带 Token，避免被他人随意调用。

### 4.3 探针接口列表

基址：`http://<前置机IP>:8088/api/customer/zaoqiang-tcm-001/msun`

| 接口 | 方法 | 说明 |
|------|------|------|
| `/env` | GET | 查看当前环境（不含密钥） |
| `/depts` | GET | 科室；建议 `?invalidFlag=-1` 查全量 |
| `/identities` | GET | 人员身份；需至少一个查询参数 |
| `/identities/sample` | GET | 自动取首个科室再查身份 |

### 4.4 curl 示例（PowerShell 可先登录再调用）

```powershell
$base = "http://127.0.0.1:8088"
$login = Invoke-RestMethod -Method Post -Uri "$base/login" -ContentType "application/json" `
  -Body '{"username":"admin","password":"admin123"}'
$token = $login.data.token
$headers = @{ Authorization = "Bearer $token" }

# 当前环境
Invoke-RestMethod -Headers $headers -Uri "$base/api/customer/zaoqiang-tcm-001/msun/env"

# 科室
Invoke-RestMethod -Headers $headers -Uri "$base/api/customer/zaoqiang-tcm-001/msun/depts?invalidFlag=-1"

# 人员（快速探针）
Invoke-RestMethod -Headers $headers -Uri "$base/api/customer/zaoqiang-tcm-001/msun/identities/sample"
```

### 4.5 HTTP 响应结构说明

成功时外层为 scminterface 统一格式，业务回参在 `data` 内：

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "requestParams": { "invalidFlag": -1 },
    "hisBody": {
      "success": true,
      "code": "0000",
      "message": "成功",
      "data": [ ... ]
    }
  },
  "tenantId": "zaoqiang-tcm-001",
  "activeEnv": "test",
  "msunBaseUrl": "https://thirdpart-graytest.msunhis.com:9443"
}
```

现场测试重点关注 **`data.hisBody`** 是否与众阳文档一致。

- [ ] 1. 使用 **方式一 + 测试库**，确认能收到 JSON（即使权限未开，也应有 `openapi@9984` 等明确错误码）

## 6. 常见问题

### Q1：`openapi@9984` 没有此产品的接口访问权限？

**原因**：当前 `appId` 未在众阳侧授权对应接口，或 URL 与文档不一致。  

**处理**：联系众阳实施工程师，提供：

- 环境（灰度/正式）、`appId`
- 接口编号：2.1.9、2.1.12
- 完整请求 URL（控制台 `>>> GET` 那一行）

### Q2：测试库成功，正式失败？

检查是否已切换到 `prod`，且正式 `hospitalId=11273002`、`orgId=11273` 与正式域名 `zqxzyyy.msuncloud.cn` 成套使用。

### Q3：人员接口报参数错误？

必须传 `roleType`、`deptId`、`identityId`、`userId` 之一。可先调 `/depts` 拿到 `deptId`，或使用 `/identities/sample`。

### Q4：探针 HTTP 返回 401？

未登录或 Token 过期。重新 `POST /login` 获取 Token。

### Q5：探针接口 404？

确认 `msun.enabled=true` 且服务已重启；URL 必须包含 `/api/customer/zaoqiang-tcm-001/msun`。
